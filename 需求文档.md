# 企业 OA 管理系统需求文档

## 1. 文档说明

### 1.1 项目定位

企业 OA 管理系统是一套面向企业内部管理人员、人事人员使用的 Web 管理后台。当前原型聚焦于三类基础主数据的维护：

- 用户管理：维护员工档案、任职信息、联系方式、学历信息。
- 部门管理：维护公司组织架构及上下级部门关系。
- 岗位管理：维护岗位配置、岗位类型及岗位所属部门。

当前版本为前端原型，数据使用前端 Mock 模拟。后续可在保持页面交互不变的基础上对接后端接口。

### 1.2 用户角色

| 角色 | 主要诉求 |
|---|---|
| HR / 人事专员 | 快速新增、编辑、查询员工档案，维护员工所属部门和岗位 |
| 部门管理人员 | 查看组织架构、岗位配置和人员归属 |
| 系统管理员 | 维护部门、岗位、用户等基础数据 |

### 1.3 设计目标

| 编号 | 目标 |
|---|---|
| G01 | 页面布局清晰，符合企业后台使用习惯 |
| G02 | 表格字段展示完整，字段内容一行展示，不换行 |
| G03 | 字段较多时通过横向滚动承载，不压缩内容、不强行省略 |
| G04 | 操作入口清晰，新增、编辑、删除、字段配置等动作易于发现 |
| G05 | 数据维护规则明确，避免删除仍被引用的部门或岗位 |

## 2. 系统范围

### 2.1 当前包含功能

| 模块 | 功能 |
|---|---|
| 用户管理 | 用户列表、用户搜索、字段配置、新增用户、编辑用户、删除用户 |
| 部门管理 | 部门树形列表、部门搜索、新增部门、添加子部门、编辑部门、删除部门 |
| 岗位管理 | 岗位列表、岗位搜索、新增岗位、编辑岗位、删除岗位 |

### 2.2 当前不包含功能

| 功能 | 说明 |
|---|---|
| 登录 / 权限控制 | 当前原型默认展示管理员身份，不区分角色权限 |
| 后端接口 | 当前使用前端 Mock 数据 |
| 分页 | 当前列表通过表格滚动展示 |
| 导入 / 导出 | 当前原型未包含 Excel 导入导出 |
| 批量操作 | 当前仅支持单条新增、编辑、删除 |

## 3. 全局页面与交互需求

### 3.1 整体布局

| 编号 | 需求 |
|---|---|
| L01 | 系统采用左侧导航 + 右侧内容区布局 |
| L02 | 左侧导航支持展开和折叠，展开宽度为 220px，折叠宽度为 72px |
| L03 | 导航菜单包含：用户管理、部门管理、岗位管理 |
| L04 | 顶部栏展示折叠按钮和当前页面标题 |
| L05 | 内容区按“概览卡片 + 筛选区 + 表格区”的结构展示 |
| L06 | 页面主色为专业蓝色，整体风格为清爽、高密度、企业后台风格 |

### 3.2 表格通用规则

| 编号 | 需求 |
|---|---|
| T01 | 表格字段内容必须一行展示，不允许自动换行 |
| T02 | 字段较多或窗口较窄时，表格区域支持横向滚动 |
| T03 | 操作列固定在右侧，方便用户在横向滚动后仍可操作 |
| T04 | 关键识别列优先靠前展示，必要时固定在左侧 |
| T05 | 表头、表格内容、状态标签在视觉上保持清晰层级 |
| T06 | 表格支持纵向滚动，最大高度根据视口高度自适应 |

### 3.3 操作反馈

| 编号 | 需求 |
|---|---|
| F01 | 新增成功后提示“新增成功” |
| F02 | 编辑保存成功后提示“更新成功” |
| F03 | 删除成功后提示“删除成功” |
| F04 | 删除前必须弹出二次确认框 |
| F05 | 因业务规则不能删除时，应给出明确提示 |

用户管理用于维护企业员工档案。页面需要支持快速筛选员工、查看完整字段、按个人习惯配置表格字段，并完成新增、编辑、删除操作。

| 指标 | 说明 |
|---|---|
| 员工总数 | 当前系统内全部员工数量 |
| 在职员工 | 状态为“在职”的员工数量 |
| 正式员工 | 用工形式为“正式员工”的员工数量 |
| 覆盖部门 | 当前员工归属过的部门数量 |

| U04 | 支持按状态筛选，状态包含：在职、离职 |

|---|---|
